Ch 2 - Managing the JBoss Application Server

All build targets create an application server configuration named admin. After the build, to run the application server, open another command window, cd to the applications server's bin directory, and enter:

./run.sh -c admin (Linux)
run -c admin (Windows)

None of the targets create a database.

Target: 01
Section: 2.2.1 & 2.2.3
Description: Creates an application server configuration with various logging configuration changes and using the system properties service.
After the build:

Search for the text "JBIA" in the server/admin/conf/jboss-log4j.xml file to locate the logging configuration changes.

To see that the logging configuration changes in action, access the example voting application at http://localhost:8080/vote. Each time you access the application a line is written to the log file. Now look in the server/admin/log directory and you should see:

  • Several server.log files, because the rolling appender was limited to 500KB and starting up the application server typically generates around 900KB of logging
  • A jbia.log file containing the logging for the example voting application

To see that the system properties service in action, change the property values in the file server/admin/conf/jbia.properties, restart the application server, and access the voting application again. The text displayed and the logging entries should reflect your changes.

Also, you can use the JMX Console to view the jboss:type=Service,name=SystemProperties MBean. Invoke the showAll() operation and look for the properties starting with "jbia".

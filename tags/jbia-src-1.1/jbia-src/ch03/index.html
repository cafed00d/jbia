Ch 3 - Deploying Applications

Some build targets create an application server configuration named deployment. After the build, to run the application server, open another command window, cd to the applications server's bin directory, and enter:

./run.sh -c deployment (Linux)
run -c deployment (Windows)

Some targets create and initialize a database named deploymentdb.

Target: 01
Section: 3.2
Description: Builds, packages and deploys several applications used to demonstrate various class loading, loader repository and class scoping concepts.
After the build:

Start the application server and open a browser at http://localhost:8080/route. Follow the instructions given by the application.

Target: 02
Section: 3.3.1
Description: Builds the jarFinder utility.
After the build:

As an example of running the jarFinder, to search for the class org.jboss.aop.advice.Interceptor within the JBoss Application Server installation directory, enter:

target/dist/jarFinder.sh $JBOSS_HOME org.jboss.aop.advice.Interceptor.class (Linux)
target\dist\jarFinder.bat %JBOSS_HOME% org.jboss.aop.advice.Interceptor.class (Windows)

Note that these instructions to run jarFinder are different than what is given in the book because we provided a script that you can use to easily run jarFinder.

This build target does not create an application server configuration.

Target: 03
Section: 3.4.1, 3.4.2
Description: Builds and deploys a data source and a Hibernate archive, and initializes the database.
After the build:

Start the application server and open a browser at http://localhost:8080/video. Enter some data and click the Add button. The bottom of the page contains a table listing all entries made so far.
